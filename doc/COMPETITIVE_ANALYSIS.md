## 🚀 Project Progress DB の独自価値（USP）

### 1. 🔴 「リスクっぽい発言」の自動検出 【競合ゼロ】

```
従来のツール:
「リスクがあります」→ 検出される
「心配しています」  → 見逃される ❌

Project Progress DB:
「心配しています」  → リスクとして検出 ✅
「厳しいかもしれない」→ リスクとして検出 ✅
「不確実な要素がある」→ リスクとして検出 ✅
```

**競合との違い**: 
- 競合は「明示的なリスク」のみ抽出
- 本製品は「曖昧な表現」からもリスクを抽出し、`source_sentence`（元発言）を保持

### 2. 📊 プロジェクトヘルススコア 【競合ゼロ】

```python
ヘルススコア = 100 - (遅延タスク減点 + 高リスク減点 + 未完了率減点 + 陳腐化減点)
```

**競合との違い**:
- 競合は個別タスク/リスクの表示のみ
- 本製品はプロジェクト全体の「健全性」を0-100で数値化

### 3. 🔄 会議間差分検出 【一部競合あり】

| 機能 | Project Progress DB | 競合 |
|------|---------------------|------|
| 新規タスク検出 | ✅ | △（一部あり） |
| ステータス変更検出 | ✅ | ❌ |
| リスクエスカレーション検出 | ✅ **独自** | ❌ |
| タスクライフサイクル追跡 | ✅ | ❌ |

### 4. 🌏 日本語ネイティブ対応

```
「来週の金曜日」 → 2024-12-13（会議日基準で計算）
「今月末まで」   → 2024-12-31
「再来週の月曜」 → 2024-12-16
```

**競合との違い**:
- 海外製品は日本語対応が弱い/翻訳精度が低い
- 本製品は日本語の自然言語日付解析に対応

### 5. 🔐 エンタープライズ対応アーキテクチャ

```
┌─────────────────────────────────────────────────────┐
│              Google Cloud Platform                  │
├─────────────────────────────────────────────────────┤
│  Cloud Run (API/Worker)  │  BigQuery (データ)       │
│  Cloud Storage (ファイル) │  Vertex AI (Gemini)     │
│  Secret Manager (認証)   │  Pub/Sub (非同期処理)   │
└─────────────────────────────────────────────────────┘
```

**競合との違い**:

- 本製品は専用GCPプロジェクトにデプロイ可能（データ主権）

## ⚠️ 競合に対する弱み・課題

### 1. 文字起こし機能がない

| 課題 | 影響 | 対策案 |
|------|------|--------|
| 手動でテキストアップロードが必要 | ユーザーの手間 | Zoom/Meet文字起こし連携を実装 |
| 会議録画機能がない | リアルタイム性がない | tl;dv等との連携API |

### 2. 会議ツール直接連携がない

| 課題 | 影響 | 対策案 |
|------|------|--------|
| Zoom/Teamsとの自動連携なし | ワークフロー断絶 | Zapier/Make連携 |
| Google Meet文字起こし未対応 | Google Workspace活用不可 | Phase 5で対応予定 |

### 3. 認知度・信頼性

| 課題 | 影響 | 対策案 |
|------|------|--------|
| 新規プロダクト | 導入障壁 | 無料トライアル提供 |
| 実績・事例がない | 信頼性 | パイロット顧客の獲得 |

## 📝 結論と推奨事項

### ✅ 差別化できている点

1. **リスクっぽい発言の自動検出** - 競合ゼロ、明確な差別化
2. **プロジェクトヘルススコア** - PMOにとって非常に価値が高い
3. **会議間差分検出** - 変化の追跡は他製品にない機能
4. **日本語ネイティブ対応** - 海外製品の弱点を突く
5. **データ主権** - 日本企業のセキュリティ要件に対応

### ⚠️ 強化が必要な点

1. **文字起こし連携** - 最大の参入障壁、優先的に対応が必要
2. **会議ツール直接連携** - ユーザー体験向上のため必要
3. **事例・実績** - 導入企業の声が必要

### 🚀 推奨アクション

1. **ローンチ時のメッセージング**:
   > 「会議のリスクを見逃さない。AIがプロジェクトの健全性を可視化。」

2. **差別化ポイントの明確化**:
   - 「リスク検出」を最大のUSPとして訴求
   - 「ヘルススコア」をPMO向けにアピール

3. **パートナーシップ検討**:
   - 文字起こしツール（Otter.ai, tl;dv）との連携
   - プロジェクト管理ツール（Notion, Asana）との連携
